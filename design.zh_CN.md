# BareTcl 設計規格書 (v5.0)

## 1. 概述

### 1.1 系統目標
本項目的核心目標是開發一個高可靠、极致精簡且完全無棧化的 Tcl 脚本解釋器內核（BareTcl）。該内核專為工業級裸機（Bare-metal）微控制器（MCU）環境（如 Renesas RH850/U2A）設計，旨在提供高度可移植的動態脚本執行能力，同時確保在資源極度受限環境下的確定性運行。

### 1.2 設計願景
通过剥離對現代操作系统（OS）和標準 C 庫（Libc）的一切依賴，在極小的 SRAM 空間內構建一个具備魯棒性的脚本執行環境。系統必須通過自有的内存統御機制和狀態機模型，徹底杜絕棧溢出（Stack Overflow）及内存泄漏（OOM）風險。

---

## 2. 設計原則與物理約束

為確保在嵌入式環境中的絕對穩定性，系統必須嚴格遵守以下四項核心約束：

### 2.1 零標準庫依賴
內核編譯選項應強制開啟 `-ffreestanding -nostdlib`。嚴禁鏈接 `<stdio.h>`, `<stdlib.h>`, `<string.h>`。所有基礎字符串操作和内存管理必須由內核實現的私有庫承担。

### 2.2 靜態内存布局 (Static Arena)
系統采用單一、連續的靜態内存池（Static Arena）進行全生命周期管理。
*   **分配模型**：通過雙向游標（`p_top` 與 `t_bot`）進行資源分配，分別管理變量數據與執行上下文（Frames）。
*   **物理隔離**：禁止使用任何動態堆内存分配（`malloc`/`free`）。

### 2.3 定長數據骨架
執行上下文（`TclFrame`）與變量定義（`TclVar`）采用定長 C 結構體。所有內部引用必須使用基於物理 Arena 起始地址的**相對偏移量（Offset）**，禁止使用絕對物理内存指針，以支持内存緊湊化處理。

### 2.4 絕對無棧化執行 (Absolute Stackless FSM)
核心執行引擎 `tcl_eval` 嚴禁采用遞歸調用。系統必須實現為一個由 `while(1)` 循環和多維 `switch(state)` 驅動的有限狀態機（FSM）。所有嵌套邏輯（如命令替換、子程序調用）均通過顯式的上下文壓棧與狀態挂起實現。

---

## 3. 核心執行引擎

### 3.1 五維狀態機模型
解釋器的核心解析邏輯被扁平化為五個主要狀態：
1.  **TOKENIZE**：解析原始字符串，識別命令與參數邊界。
2.  **EXPAND**：處理變量替換（`$`）與命令替換（`[...]`）。
3.  **EXECUTE**：定位指令處理函數並執行。
4.  **RESUME**：當子結界執行完畢後，恢復父結界的執行上下文並處理返回結果。
5.  **CLEANUP**：回收臨時資源，處理異常狀態碼冒泡。

---

## 4. 内存管理與緊湊化協議 (GC)

### 4.1 内存池結構
*   **低地址區 (`p_top`)**：存儲變量名、變量值及 `TclVar` 結構體。
*   **高地址區 (`t_bot`)**：存存储執行棧幀（`TclFrame`）及調用鏈。

### 4.2 緊湊化垃圾回收 (Compacting GC)
1.  **觸發機制**：當無法分配連續空間時，觸發同步 GC。
2.  **標記階段**：深度遍歷全局變量表及當前執行鏈，標記所有可達對象。
3.  **壓縮階段**：執行物理地址平移（Slide Compacting），將存活數據向 Arena 起始地址擠壓。
4.  **重定位**：同步更新所有 `TclVar` 內部的偏移量，確保引用一致性。

## 5. 原初指令集 (Atomic Set)

內核僅實現以下 18 個原子指令作為自舉基礎。

| 指令 | 描述 |
| :--- | :--- |
| `set` | 變量讀寫，嚴格存在性校驗。 |
| `proc` | 命令注冊，定義新作用域。 |
| `if` / `while` | 基礎控制流。 |
| `expr` | 算術與邏輯運算（支持延遲求值）。 |
| `return` / `error` | 終止與狀態冒泡。 |
| `break` / `continue` | 循環控制。 |
| `eval` / `catch` | 動態執行與異常捕獲。 |
| `upvar` / `uplevel` | 跨作用域訪問。 |
| `list` / `lindex` | 列表封裝與零分配解析。 |
| `llength` / `lrange` | 列表長度與切片。 |
| `unset` | 變量銷毀。 |

## 6. 工程化與質量控制

### 6.1 內核純淨度校驗
`build.sh` 通過以下流水線確保 Libc-Free：
*   獨立編譯內核並使用 `nm -u` 檢查符號。
*   **准入標準**：輸出必須為空。若引用外部符號（如 `memset`），構建失敗。

### 6.2 工業級驗證
通過以下嚴苛測試：
*   遞歸算法（斐波那契、漢諾塔、8皇后）。
*   GC 壓力測試（64KB 空間內萬級對象 churn）。

---

## 7. 基本類型系統

嚴禁使用 C 原生類型，統一使用固定寬度類型：
*   `tcl_u8`: 字符/字節。
*   `tcl_i32`: 默認整數。
*   `tcl_u32`: 偏移量/計數。

---

## 8. 結論
BareTcl 通過物理 Arena、無棧化 FSM 和 18 條原子法則，構建了一個工業級的執行壁壘。
