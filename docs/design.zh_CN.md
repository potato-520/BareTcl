# BareTcl 设计规格书 (v5.0)

## 1. 概述

### 1.1 系统目标
本项目的核心目标是开发一个高可靠、极致精简且完全无栈化的 Tcl 脚本解释器内核（BareTcl）。该内核专为工业级裸机（Bare-metal）微控制器（MCU）环境设计，确保在资源极度受限环境下的确定性运行。

---

## 2. 物理约束

### 2.1 零标准库依赖
内核编译强制开启 `-ffreestanding -nostdlib`。严禁链接任何标准 C 库。

### 2.2 静态内存布局 (Static Arena)
系统采用单一连续的静态内存池。
*   **分配模型**：通过双向游标管理变量数据与执行上下文（Frames）。
*   **物理隔离**：禁止使用 `malloc`/`free`。

### 2.3 绝对无栈化执行
核心引擎 `tcl_eval` 严禁递归。所有嵌套逻辑均通过显式的上下文压栈实现。

---

## 3. BareTcl Shell 组件架构

为了在裸机环境下提供高级交互能力，BareTcl 集成了一个专门的**行编辑外壳 (Shell)**。

### 3.1 架构设计
Shell 作为一个独立的有限状态机实现 (`src/baretcl_shell.c`)，负责解析来自串口的原始字节。它在不使用动态内存的情况下维护自身的行缓冲区和历史记录。

### 3.2 ANSI 转义序列状态机
Shell 解析标准 ANSI 转义序列以提供专业 CLI 特性：
*   **导航**：支持上下键切换命令历史，左右键移动行内光标。
*   **编辑**：支持 Backspace 退格删除，并同步更新终端显示。
*   **智能多行**：实时跟踪花括号 `{}` 嵌套层级，自动在“直接执行”与“多行续行”模式间切换。

---

## 4. 内存管理与 GC

### 4.1 紧凑化垃圾回收 (Compacting GC)
1.  **标记**：深度遍历全局与局部变量，标记可达对象。
2.  **压缩**：执行物理地址平移，消除内存空洞。
3.  **重定位**：基于相对偏移量更新所有内部引用。

---

## 5. 原子指令集
内核仅实现 18 条原子指令（如 `set`, `proc`, `if`, `while`, `eval`, `catch`）。高级功能由 Tcl 自举实现。

---

## 6. 工程化与质量控制
*   **纯净度校验**：通过 `nm -u` 确保内核无外部符号引用。
*   **工业验证**：通过 8 皇后算法、深层递归及 GC 极限压力测试。

---

## 7. 项目组织结构

| 目录/文件 | 说明 |
| :--- | :--- |
| `src/` | **核心引擎**：包含状态机、扩展指令、Shell 组件及自举源码。 |
| `tests/` | **验证套件**：包含工业级全量测试脚本。 |
| `tools/` | **构建工具**：用于脚本静态化编译。 |
| `examples/` | **集成示例**：展示 Linux 环境下的终端 Raw Mode 集成。 |
| `docs/` | **文档中心**：多语言设计规格书及用户手册。 |

---

## 8. 结论
BareTcl 通过物理 Arena、无栈化 FSM 和集成式智能 Shell，构建了工业级脚本执行的稳态环境。
