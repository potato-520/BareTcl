# BareTcl 設計仕様書 (v5.0)

## 1. 概要

### 1.1 システム目標
このプロジェクトの核心目標は、高信頼、超軽量、かつ完全にスタックレスな Tcl スクリプトインタプリタコア（BareTcl）を開発することです。産業グレードのベアメタル MCU 環境向けに特別に設計されています。

---

## 2. 物理的制約

### 2.1 Libc 依存ゼロ
コアのコンパイルには `-ffreestanding -nostdlib` を強制します。標準 C ライブラリへのリンクは厳禁です。

### 2.2 静的 Arena メモリ配置
単一の連続した静的メモリプールを使用します。
*   **割り当てモデル**: `p_top` と `t_bot` の双方向カーソルにより管理。
*   **物理的隔離**: `malloc`/`free` の使用は一切禁止されています。

### 2.3 絶対的スタックレス実行
実行エンジン `tcl_eval` 内での再帰呼び出しは厳禁です。すべての入れ子構造は明示的なコンテキストのスタック操作によって実現されます。

---

## 3. BareTcl Shell コンポーネント

ベアメタル環境で高度な対話機能を提供するため、BareTcl には専用の**ラインエディタ・シェル**が組み込まれています。

### 3.1 アーキテクチャ
シェルは独立した状態機 (`src/baretcl_shell.c`) として実装され、UART からの生バイトを処理します。動的メモリ割り当てを行わずに、独自のラインバッファと履歴を維持します。

### 3.2 ANSI エスケープ状態機
標準 ANSI エスケープシーケンスを解析し、プロフェッショナルな CLI 機能を提供します：
*   **ナビゲーション**: 上下キーによる最大 16 件のコマンド履歴の切り替え、左右キーによる行内カーソル移動。
*   **編集**: バックスペース処理とターミナル表示の同期。
*   **インテリジェント複数行入力**: 花括弧 `{}` のネストを追跡し、直接実行モードと継続入力モードを自動的に切り替えます。

---

## 4. メモリ管理と GC

### 4.1 コンパクト GC
1.  **マーキング**: 到達可能なすべてのオブジェクトを走査。
2.  **コンパクト化**: 生存データを Arena の先頭へ物理移動。
3.  **再配置**: 相対オフセットに基づき内部参照を一括更新。

---

## 5. 原子命令セット
コアは 18 個の原子命令のみを実装。高度な制御は Tcl レベルで構築されます。

---

## 6. プロジェクト構成

| ディレクトリ/ファイル | 説明 |
| :--- | :--- |
| `src/` | **コアエンジン**: `tcl_core.c`, `extcmd.c`, `baretcl_shell.c`, `tcllib.tcl` |
| `tests/` | **検証スイート**: 産業用ストレステストスクリプト。 |
| `tools/` | **ビルドツール**: ブートストラップ用コンパイルスクリプト。 |

| `docs/` | **ドキュメント**: 多言語仕様書およびユーザーガイド。 |

---

## 7. 結論
BareTcl は、物理的な Arena メモリ、スタックレス FSM、および統合されたスマートシェルを通じて、産業用実行環境を提供します。
