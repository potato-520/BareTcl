# BareTcl

**世界最小クラスの産業グレード Tcl インタプリタコア。**

BareTcl は、ルネサス RH850 や ARM Cortex-M などの非常に厳しいベアメタル (Bare-metal) MCU 環境向けに設計された、超軽量・スタックレス・Libc フリーの Tcl インタプリタです。実行の決定論、メモリ安全性、およびアーキテクチャの純粋性を最優先しています。

[English](./README.md) | [中文版](./README.zh_CN.md)

---

## 主な特徴

- **産業グレードの信頼性**: ミッションクリティカルなシステム向けに構築されたゼロ依存ポリシー。
- **スタックレス FSM**: 完全に非再帰的な実行エンジン。C スタックオーバーフローのリスクを排除します。
- **最小 18 命令セット**: C カーネルに実装された命令はわずか 18 個。`for` や `incr` などの高度な制御は Tcl 自身によるセルフブートストラップで実装されています。
- **コンパクト GC**: Arena 方式の移動型ガベージコレクタにより、メモリ断片化（フラグメンテーション）をゼロに抑えます。
- **Libc 完全依存排除**: 標準 C ライブラリを一切使用しません。`-ffreestanding -nostdlib` でコンパイル可能です。
- **固定幅アーキテクチャ**: `tcl_i32` や `tcl_u8` を厳格に使用し、プラットフォームを超えたビットレベルの決定論を保証します。

---

## 実証された堅牢性とパフォーマンス

BareTcl は、単なる軽量なコアではなく、複雑なアルゴリズムにも対応できる高い能力を備えています：

- **完全なセルフブートストラップ**: `for`、`foreach`、`incr` などの標準的な命令は Tcl 自身で記述され、ビルド時に C カーネルへ統合されます。
- **8クイーン問題の解決**: 複雑なリスト操作と深い再帰を必要とする 8 クイーン問題をベアメタル環境でクリアし、論理的な完成度を証明しました。
- **過酷な GC ストレステスト**: 非常に限られたメモリ空間で数万個のオブジェクトを生成・破棄するテストをパス。メモリリークや断片化は一切発生しません。
- **実行効率**: MCU のサイクル効率を最大限に高めるために最適化された FSM ループにより、低速なハードウェア上でも軽快に動作します。

---

## アーキテクチャの純粋性

BareTcl は "The Great Purge"（大浄化）哲学に従っています：
1. **コア (Core)**: 言語を構築するために必要な最小限の 18 個の原子命令のみを保持。
2. **拡張 (Extensions)**: プラットフォーム依存のロジック（I/O、OS 呼び出し）は `extcmd.c` に厳格に分離。
3. **ブートストラップ (Bootstrap)**: 高度なコマンドは Tcl で記述され、ビルドプロセスを通じて C カーネルに統合されます。
