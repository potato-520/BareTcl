# BareTcl

**世界最小クラスの産業グレード Tcl インタプリタコア。**

BareTcl は単なるインタプリタではありません。埋め込みエンジニアのための**「究極の武器」**です。退屈で時間のかかるシリアル通信のプロトコル解析や、バグの温床となりやすい独自の状態機の実装からあなたを解放します。BareTcl は、ベアメタルシステムに「ゼロコスト」で動的なスクリプト環境を提供します。

## 「ゼロ・要求」哲学

BareTcl は、移植が世界で最も容易なインタプリタとして設計されています。システム環境に対して**何も要求しません**：

- **`malloc` / `free` 不要**：固定サイズの静的 Arena 方式を採用。ヒープの断片化リスクはゼロです。
- **`libc` 不要**：完全に独立した C コード。`<stdio.h>` や `<string.h>` などの標準ライブラリを一切必要としません。
- **C スタック依存なし**：完全なスタックレス FSM アーキテクチャ。Tcl 側の再帰がどれほど深くても、C のスタックオーバーフローは**絶対に**発生しません。
- **`setjmp` / `longjmp` 不要**：複雑なジャンプメカニズムに頼らない、決定論的なフロー制御。
- **ビルドも極めて単純**：シングルファイルのコアで、`-ffreestanding -nostdlib` でのコンパイルに対応。

## なぜ BareTcl なのか？（退屈な開発からの解放）

数十年にわたり、埋め込み開発者はシリアルコマンド解析ルーチンの作成という「退屈な」作業に縛られてきました。BareTcl はその時代を終わらせます。

BareTcl を統合することで、通常のシリアルポートを強力な動的コンソールへと変貌させることができます。すべての機能をハードコードする代わりに、実行時に複雑なロジックやループ、条件判断を直接実行できるようになります。これは MCU を真にプログラマブルな力へと変える**「神兵利器（至高の武器）」**です。

---

## 主な特徴

- **産業グレードの信頼性**: ミッションクリティカルなシステム向けに構築されたゼロ依存ポリシー。
- **最小 18 命令セット**: カーネルはわずか 18 個の原子命令で構成。`for` や `foreach` などの高度な制御は Tcl 自身によるセルフブートストラップで実装。
- **コンパクト GC**: Arena 方式の移動型ガベージコレクタにより、数万個のオブジェクト生成後もメモリ断片化をゼロに維持。
- **ビットレベルの決定論**: `tcl_i32` や `tcl_u8` を厳格に使用し、あらゆるプラットフォームで同一の挙動を保証。

---

## 実証された堅牢性とパフォーマンス

- **完全なセルフブートストラップ**: 命令セットが完備されているため、言語自身が自分自身のライブラリを構築します。
- **8クイーン問題の解決**: ベアメタル環境で複雑なリスト操作と深い再帰をクリアし、論理的完成度を証明。
- **過酷なストレステスト**: わずか 64KB の Arena 空間で数万回の変数 churn テストをパス。リークなし、断片化なし。

## 開発者ガイド

### 1. ベアメタルへの移植
1. **HAL の実装**: `void tcl_hal_puts(const tcl_u8 *s)` を提供し、出力を UART 等にマッピングします。
2. **Arena の初期化**: `tcl_init(buffer, size)` を呼び出します。
3. **自挙ライブラリのロード**: `tcl_load_bootstrap(ctx)` を呼び出し、高度な Tcl 機能を有効化します。

### 2. C 言語による拡張
```c
static tcl_i32 my_hardware_cmd(TclCtx *ctx, tcl_i32 argc, tcl_u32 *argv) {
    // ここにハードウェア制御ロジックを記述...
    return TCL_OK;
}
tcl_register_c_cmd((const tcl_u8 *)"hw_ctrl", my_hardware_cmd);
```
