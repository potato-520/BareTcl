# BareTcl

**世界で最もコンパクトな産業グレード Tcl インタプリタ・コア**

BareTcl は単なるインタプリタではありません。組み込み開発者のための**強力な武器**です。煩雑なシリアル通信ロジック、手動のプロトコル解析、そして脆弱なアドホック状態マシンからあなたを完全に解放するために設計されました。BareTcl は、ベアメタルシステムに対して「ゼロコスト」に近い動的スクリプト統合体験を提供します。

## 「究極のゼロ依存」哲学

BareTcl は、あなたが出会う中で最もポータビリティの高いインタプリタです。システム環境に対して**ほとんど何も要求しません**：

- **`malloc` / `free` 不要**：固定サイズの静的アリーナメモリプールに完全に基づいています。ヒープ断片化のリスクを徹底的に排除します。
- **`libc` 不要**：純粋で独立した C コード。`<stdio.h>` や `<string.h>`、あるいはいかなる標準ライブラリのヘッダーにも依存しません。
- **C スタックサポート不要**：完全にスタックレス（Stackless）な状態マシンアーキテクチャ。Tcl スクリプト内の再帰がどれほど深くても、C 言語のコールスタックオーバーフローを引き起こすことは**決してありません**。
- **`setjmp` / `longjmp` 不要**：複雑なジャンプ機構に依存しない、決定論的な制御フロー。
- **複雑なビルド不要**：単一ファイルのコア。`-ffreestanding -nostdlib` でのコンパイルをサポートします。

## なぜ BareTcl を選ぶのか？（退屈な開発からの脱却）

数十年にわたり、組み込み開発者は退屈なシリアルコマンド解析器を繰り返し書くことを強いられてきました。BareTcl はこの現状を終わらせます。

BareTcl を統合することで、普通のシリアルポートを強力な動的コンソールに変えることができます。すべての機能に対して解析ロジックをハードコードする必要はなく、実行時に複雑な Tcl ロジック、ループ、条件判断を直接実行できます。これは真の**組み込みスイスアーミーナイフ**であり、静的な MCU に瞬時に柔軟なプログラミング能力を与えます。

## 主な特徴

- **産業グレードの信頼性**：ミッションクリティカルなシステムのために構築され、厳格なゼロ依存戦略を貫いています。
- **インテリジェントな対話型シェル (Shell)**：バックスペース、矢印キーによるカーソル移動、直近 16 件の履歴切り替え、インテリジェントな複数行入力（閉じられていない中括弧の自動識別）をサポートする軽量な行エディタを内蔵。
- **原子 18 命令コア**：C コアには 18 の原子命令のみが含まれています。すべての高度なロジック（`for`, `incr`, `foreach` など）は、Tcl スクリプトによる自己ブートストラップで実現されています。
- **コンパクト化 GC**：移動・コンパクト化ガベージコレクタを採用し、万単位のオブジェクト入れ替えが発生してもアリーナ空間を断片化ゼロに保ちます。
- **ビットレベルの決定論**：`tcl_i32`, `tcl_u8` などの固定幅型を厳格に使用し、プラットフォーム間で一貫した動作を保証します。

---

## 強力なパフォーマンスと信頼性の検証

- **自己ブートストラップの完全性 (Self-Bootstrap)**：核心命令セットは高度に完備されており、標準ライブラリは完全に Tcl 自身によって構築され、静的に統合されています。
- **8 クイーン問題の解法 (8-Queens)**：ベアメタル環境で 8 クイーンアルゴリズムを完璧に実行し、深い再帰と複雑なリストを処理する能力を証明しました。
- **GC 極限ストレス試験**：わずか 64KB のアリーナ空間で数万回の変数入れ替えテストに耐え、リークゼロ、断片化ゼロを達成しました。

## 開発者ガイド

### 1. ベアメタル移植の手順
1. **HAL 層の実装**：シンプルな `void tcl_hal_puts(const tcl_u8 *s)` を提供。
2. **アリーナの初期化**：`tcl_init(buffer, size)` を呼び出す。
3. **シェルの駆動**：シリアルポートから受信した各バイトを `shell_handle_char(&sh, byte, "> ")` に渡す。
4. **解析と実行**：シェル関数が `1` を返したとき、`sh.line` を `tcl_eval` に渡して実行。

### 2. C 言語による拡張
```c
static tcl_i32 my_hardware_cmd(TclCtx *ctx, tcl_i32 argc, tcl_u32 *argv) {
    // ここでハードウェアを直接制御するロジックを実装します...
    return TCL_OK;
}
tcl_register_c_cmd((const tcl_u8 *)"hw_ctrl", my_hardware_cmd);
```
