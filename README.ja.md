# BareTcl 🚀

**世界最小クラスの産業グレード Tcl インタプリタコア。**

BareTcl は、ルネサス RH850 や ARM Cortex-M などの非常に厳しい**ベアメタル (Bare-metal)** MCU 環境向けに設計された、超軽量・**スタックレス**・**Libc フリー**の Tcl インタプリタです。実行の決定論、メモリ安全性、およびアーキテクチャの純粋性を最優先しています。

[English](./README.md) | [中文版](./README.zh_CN.md)

---

## 🌟 主な特徴

- **🛡️ 産業グレードの信頼性**: ミッションクリティカルなシステム向けに構築されたゼロ依存ポリシー。
- **🏗️ スタックレス FSM**: 完全な非再帰実行エンジン。C スタックオーバーフローの心配がありません。
- **💎 最小 18 命令セット**: C カーネルに実装された命令はわずか 18 個。`for` や `incr` などの高度な制御は Tcl で実装されています。
- **🧠 コンパクト GC**: Arena 方式の移動型ガベージコレクタにより、メモリ断片化（フラグメンテーション）をゼロに抑えます。
- **🔌 Libc 完全依存排除**: 標準 C ライブラリを一切使用しません。`-ffreestanding -nostdlib` でコンパイル可能です。
- **⚡ 固定幅アーキテクチャ**: `tcl_i32` や `tcl_u8` を厳格に使用し、プラットフォームを超えたビットレベルの決定論を保証します。

---

## 🏗️ アーキテクチャの純粋性

BareTcl は **"The Great Purge"（大浄化）** 哲学に従っています：
1. **コア (Core)**: 言語を構築するために必要な最小限の 18 個の原子命令のみを保持。
2. **拡張 (Extensions)**: プラットフォーム依存のロジック（I/O、OS 呼び出し）は `extcmd.c` に厳格に分離。
3. **ブートストラップ (Bootstrap)**: 高度なコマンドは Tcl で記述され、C カーネルにクロスコンパイルされます。

## 🚀 クイックスタート

```bash
# 必要要件: gcc, python3
bash build.sh
```

```bash
./tclsh
> set x 10
> puts [expr $x * 2]
20
```
