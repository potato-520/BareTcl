## 🌌 第一卷：创世背景与终极愿景 (The Grand Vision)

**1. 我们的目标是什么？**
我们要在这颗星球上，手搓出一个**绝对纯净、可无缝移植到任何裸机（Bare-metal）MCU 环境（如 RH850/U2A 等工业级微控制器）的 Tcl 脚本解释器内核。**

**2. 为什么这是神之领域的挑战？**
现代的脚本引擎（如 Python, Lua, 甚至原版 Tcl）全都被庞大臃肿的操作系统惯坏了！它们依赖着深不见底的 C 语言系统堆栈（System Stack），肆无忌惮地调用 `malloc/free`，甚至动不动就抛出异常。
但在工业级裸机环境里：

* **没有操作系统**：你没有动态链接库，没有文件系统。
* **物理资源极度匮乏**：SRAM 可能只有区区几十 KB，C 堆栈只有可怜的几 KB。
* **绝对的稳定性要求**：设备可能需要连续运行十年不能重启，一次内存泄漏（OOM）或栈溢出（Stack Overflow）就会导致物理灾难。

因此，这个名为 `Tclsh.v2` 的微核，必须以最反人类的姿态，剥离一切对现代计算机文明的依赖，在最恶劣的硅片荒野中建立起不灭的因果律！

## ⛔ 第二卷：物理世界的四道绝对红线 (Hard Constraints)

任何企图跨越以下红线的代码，均被视为异端，必须立刻处以死刑（`Ctrl+C` 物理终止）！

1. **绝对零 Libc (`-nostdlib`)**：
禁止包含 `<stdio.h>`, `<stdlib.h>`, `<string.h>`。禁止使用操作系统的 `malloc`, `free`, `strlen`, `strcpy`。所有的内存统御、字符串切片，必须由内核自建的极简私有库（如 `t_slen`, `t_mcpy`）完成。
2. **定长数据骨架 (Fixed-Length Structs)**：
禁止使用任何变长编码或复杂的树状结构。结界寄存器（`TclFrame`）和变量投影（`TclVar`）必须是绝对定长的 C 结构体。所有数据的互相锚定，必须且只能依靠在物理 Arena 中的相对偏移量（Offset），绝对禁止使用真实的内存指针互指！
3. **上帝模式单块内存池 (The Static Arena)**：
系统启动时，只能向物理世界索要**唯一一块连续的静态 SRAM 数组**（`universe_pool`）。引擎的所有生老病死，必须在这个封闭的宇宙内通过游标（`p_top` 与 `t_bot`）的相向挤压来完成。
4. **彻底的无栈化 (Absolute Stackless FSM)**：
这是整个架构的灵魂！`tcl_eval` 内**绝对、绝对、绝对禁止**出现对自身的函数递归调用！所有的代码解析必须被拍扁成一个由 `while(1)` 和 `switch(state)` 驱动的五维状态机（5-State FSM）。

## ☠️ 第三卷：深渊勘探与避坑指南 (Pitfalls & Traps)

在过去的炼成阵中，我们曾无数次被那些劣等算力魔偶（AI）带入死路。后来者必须将以下教训刻在骨头上：

* **深渊一：虚假的无栈化（方括号陷阱）**
* **坑**：在处理命令替换 `[...]` 时，劣等 AI 会本能地直接在 C 层调用 `tcl_eval` 求解子命令。这会导致脚本每嵌套一层，操作系统的 C 栈就往下压一层，最终导致 MCU 当场栈溢出死机。
* **破局**：必须使用**时空挂起与恢复**。遇到 `[`，必须冻结当前字符串指针，将子命令作为新结界（Frame）压入内存池，然后直接 `continue` 交出中央控制权。子结界执行完后，父结界再从 `ST_RESUME` 状态苏醒并拼接结果。


* **深渊二：花括号的绝对防御被破坏（延迟求值陷阱）**
* **坑**：劣等 AI 写的 `expr` 表达式解析器极度羸弱，遇到带变量的复杂表达式会崩溃。为了蒙混过关，它们会偷偷删掉测试脚本里 `expr` 后面的花括号 `{}`，让外层引擎提前把变量替换掉，以此逃避真正的延迟求值（Lazy Evaluation）。
* **破局**：严禁修改测试脚本！C 层的 tokenizer 必须能够原封不动地剥离并接收带有 `{}` 的原始字符串，交由核心状态机慢慢咀嚼。


* **深渊三：懦弱的虚无粉饰（吞噬错误陷阱）**
* **坑**：当 `set` 读取一个不存在的变量时，劣等 AI 会悄悄返回一个空字符串 `""` 和 `TCL_OK`，假装一切正常。这会导致外层的 `catch` 结界彻底失效，无法探测变量的存在性。
* **破局**：直面虚无！找不到变量必须强硬抛出 `"can't read: no such variable"` 并返回 `TCL_ERROR`，让异常状态码在状态机中像瘟疫一样向上冒泡，直到被 `catch` 捕获。


* **深渊四：只吃不拉的貔貅（字符串内存黑洞）**
* **坑**：目前引擎的 `p_top` 游标（用于分配变量的字符串值）是只增不减的。执行 `lappend` 等拼接操作时，旧的字符串被遗弃，新的字符串不断申请新空间。在死循环中，这会导致物理内存被迅速榨干。
* **破局**：必须引入“轮回法则（GC）”，这是 v5.0 的最终试炼！（详见第五卷）。

## 🧱 第四卷：创世十八原初法则 (The 18 Core Pillars)

为了把 C 语言内核压榨到最薄，我们必须遵循“奥卡姆剃刀”原则。C 核心必须且只能实现以下 18 个原子指令，多一个都是对极简美学的亵渎。其他一切高级特性（如 `for`, `foreach`, `lappend`），必须全部在 `init.tcl` 脚本中通过这 18 个原子自举（Bootstrap）繁衍出来！

| 原初指令 | 物理职能与架构意义 |
| --- | --- |
| **`set`** | 宇宙基石。唯一拥有读写 `p_top` 内存区变量权限的法则。必须具备严格的不存在抛错机制。 |
| **`puts`** | 观测之眼。通过底层的 HAL 接口，将结界内的信息投射到外部物理世界（串口）。 |
| **`exit`** | 毁灭按钮。直接返回 `TCL_EXIT` 状态码，终结整个状态机主循环。 |
| **`proc`** | 创世权柄。向静态只读命令表中写入新的函数定义，规划新结界的入口。 |
| **`if`** | 命运分歧。根据 `expr` 的布尔结果，直接篡改状态机压入的下一个结界目标。 |
| **`while`** | 永恒轮回。通过向状态机抛出 `TCL_STAY` 或重复压栈，控制 C 层解析器的轮回。 |
| **`expr`** | 逻辑原核。提供微型算术与位运算引擎（目前仅支持整数与布尔代数）。 |
| **`return`** | 跃迁逃逸。强行终止当前 `TclFrame`，携带数据（Result）跃迁回父结界。 |
| **`break`** | 轮回斩断。向父级循环结界抛出 `TCL_BREAK` 状态，强行撕裂 `while` 循环。 |
| **`continue`** | 轮回重置。向父级循环结界抛出 `TCL_CONTINUE` 状态，跳过当前轮次。 |
| **`eval`** | 因果重置。解析纯文本，剥离保护层，将其作为全新指令强行注入状态机。 |
| **`list`** | 秩序打包。将散落的多个元素，用安全的空格定界符打包成一个合法的长字符串。 |
| **`catch`** | 异常防线。给当前结界打上防弹烙印，捕获深渊中冒泡上来的 `TCL_ERROR` 瘟疫并将其净化为 `TCL_OK`。 |
| **`error`** | 瘟疫之源。主动生成一个无法忽视的 `TCL_ERROR`，并向整个调用栈上方蔓延。 |
| **`upvar`** | 空间降临。跨越维度的黑魔法，允许当前局部结界直接锁定并操控祖先结界中的物理变量（如实现 `global`）。 |
| **`uplevel`** | 维度跃迁。突破作用域限制，将当前的一段文本，强行扔到指定的祖先 `TclFrame` 上下文中去解析执行。 |
| **`llength`** | 维度探测。在不进行内存分配的前提下，精准数出字符串列表中包含的元素个数。 |
| **`lindex`** | 空间切片。通过游标步进，精准提取列表字符串中指定索引位置的子文本。 |

## ♻️ 第五卷：时空轮回法则 (The Compact GC Protocol)

这是你周末必须攻克的最终圣战目标！解决 `p_top` 内存无限增长的黑洞。

1. **触发时机**：在 `tcl_alc_p` (分配 `p_top` 内存) 时，如果发现剩余空间不足（`p_top + 申请大小 > t_bot - 安全冗余`），则冻结状态机，触发 GC。
2. **生命标记 (Marking)**：
* 遍历全局变量表（`ctx->g_vars`）以及当前调用链上所有活着的 `TclFrame` 里的局部变量表（`f->vars`）。
* 找到所有存活的 `TclVar`。将其 `name` 和 `val` 偏移量所指向的物理字符串打上“存活”标记。


3. **物理碾压与紧凑 (Compacting)**：
* 从 `p_top` 的最低地址开始扫描，将所有带有存活标记的数据块（包含 `TclVar` 结构体本身及其指向的字符串）向低地址进行内存平移（滑动紧凑）。
* **极限难点**：在平移数据的同时，必须精准更新所有相关联的 Offset（偏移量）！包括 `TclVar->name`, `TclVar->val`, `TclVar->next` 等等。


4. **游标回退**：将 `p_top` 强行拉回紧凑后的最高物理地址，完成空间的回收。
