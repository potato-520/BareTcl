# Tclsh.v2 - 産業グレードの不滅な組み込みスクリプトエンジン

Tclsh.v2は汎用のTclインタプリタではありません。これは、**産業用ベアメタルマイクロコントローラ（MCU）**のために設計された、極めて高性能で完全に制御された、理論上「実行の不死性」を持つコアです。Renesas RH850/U2Aなどの過酷な環境における極限の探索から誕生し、64KB SRAMのような極めてリソースが制限された状況下で、絶対的な確定性を持つ動的なスクリプト実行能力を提供することを目指しています。

## コア哲学：3つの至高原則

本プロジェクトは『Tclsh.v2 創世法典』の中核的なレッドラインを厳格に遵守し、論理的な越えられない壁を構築しています：

1. **確定的な実行 (Deterministic Runtime)**：すべてのメモリ割り当て、命令ジャンプ、状態遷移はO(1)または制御されたO(N)の計算量であり、予測不可能な再帰や動的なヒープ割り当ては厳禁です。
2. **絶対的な物理的独立性 (Total Purity)**：コアは **100% Libc-Free** です。`<stdio.h>`、`<stdlib.h>`、`<string.h>`、`<stdint.h>` を含みません。すべての基礎型の定義、メモリカーソル管理、文字列処理、数学演算は手作業で実装され、1バイトのロジックまで完全に制御下にあることを保証します。
3. **スタックレスな永続性 (Stackless Immortality)**：5次元の有限状態マシン（FSM）を利用して、C言語ネイティブのコールスタックを代替します。数万層の深さの再帰アルゴリズムを実行しても、Cスタックの消費量は常に一定です。「輪廻の法則」（GC）と組み合わせることで、物理メモリが枯渇した際、システムは自動的に収縮・再構築を行い、無期限の継続稼働を実現します。

## 深度テクニカル実装

### 1. 5次元スタックレスFSM (Backbone)
コアは評価ロジックを5つの核となる状態（結界）に分解します：
*   **TOKENIZE**：物理レベルの文字ストリーム解析。入れ子になった中括弧 `{}` と角括弧 `[]` の原子的な切り出しをサポートします。
*   **EXPAND**：因果律の展開。親フレームを破壊することなく、変数置換 (`$`) と再帰的なコマンド置換 (`[...]`) をその場で処理します。
*   **EXECUTE**：原子命令の投影。18の根源的原子命令またはカスタムプロシージャを呼び出します。
*   **RESUME**：チェックポイントからの覚醒。サブフレームの実行完了後、実行コンテキストを正確に埋め戻し、親フレームを復元します。
*   **COND/LOOP**：論理ループ。Cのループではなく状態マシンの回帰を通じて、`while` の永続的な実行を実現します。

### 2. 輪廻の法則 (Compacting GC)
単調増加するメモリカーソルがアリーナを枯渇させるのを防ぐため、コアには厳密な **Mark-Compact プロトコル** が実装されています：
*   **ObjHeader アドレス指定**：各ヒープオブジェクトは8バイトのメタデータを持ち、線形な物理トラバースをサポートします。
*   **多重アンカー保護 (Rooting)**：`tmp_roots` 配列を通じて生成途中の中間変数を保護し、「未生先死」のGCトラップを回避します。
*   **物理アドレスの再配置**：追加のポインタ配列を使用せずに4段階の緊密化アルゴリズムを実行し、すべてのオフセット参照を動的に修正します。

## 18の根源的原子命令

現在、以下のコア命令が完全に実装されています：
*   `set`：変数次元の定義と取得。
*   `puts`：HAL層を通じて外部世界へ情報を投影。
*   `expr`：三元整数算術演算と論理比較。
*   `if` / `while`：高次元の論理分岐とループ。
*   `proc`：新しい実行結界（プロシージャ）の創造。
*   `incr`：変数の単方向進化。
*   `return`：結界エネルギーの回帰。
*   `exit`：インタプリタ宇宙の完全な終焉。

## 開発者への警告

*   **安全レッドライン**：エージェントモードでのすべてのテスト実行は、スクリプトの論理エラーによるハングアップを防ぐため、`timeout 10s` を前置しなければなりません。
*   **アライメント要求**：すべてのメモリ割り当ては、高性能MCUのバスアクセス要件に合わせるため、厳格に8バイトアライメントで実行されます。
